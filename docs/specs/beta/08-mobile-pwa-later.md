# Feature: Mobile and PWA Later

## Status

**Deferred** — responsive web exists; no PWA manifest, service worker, or native app.

## Implementation Snapshot

### Current (partial responsive web)

- Dashboard and settings use Tailwind responsive utilities (`p-4`, `max-w-*`, flex layouts).
- Primary flows work in mobile viewport browsers: setup, login, library select, filter, spin, result.
- Touch targets on main spin button are reasonably sized; formal 44px audit not completed.
- Plex deep links (`playLinks.ios`, `playLinks.android`) generated by backend for mobile app handoff.
- No `public/manifest.json`, no service worker, no install prompt metadata.

### Deferred

- PWA installability (manifest, icons, service worker).
- Offline behavior and sync queue.
- React Native / Expo native app.
- Shared TypeScript API package for mobile client.

### Target / Future

Mobile/PWA should consume existing REST API without schema changes until offline sync is explicitly planned.

## 2. API Contract

### Current — use existing endpoints

| Domain | Endpoints |
|--------|-----------|
| Auth | `/api/settings/status`, `/api/auth/login`, `/api/auth/me`, `/api/auth/logout` |
| Library | `/api/library/**` |
| Selection | `/api/selection/pool-count`, `/api/selection/random` |
| Watched | `/api/watched/**`, `/api/tautulli/**` |
| Settings | `/api/settings`, `/api/users/me/preferences` |
| Spin history | `/api/spin-history/**` |

All require session cookie (`credentials: 'include'`). Native apps must use secure cookie or token strategy — **not defined yet**.

### Future optional endpoints

| Method | Path | Purpose |
|--------|------|---------|
| GET | `/api/sync/changes?since=ISO` | Changed preferences/watched/metadata |
| POST | `/api/sync/queue` | Apply offline client mutations |

## 5. Edge Cases

- Plex app deep links differ iOS vs Android vs web fallback.
- Mobile browser blocks custom URL schemes.
- Large posters/animations on low-end devices.
- Session expires while PWA installed.
- Home-network Plex server unreachable off LAN.
- `SECURE_COOKIES` env affects cookie behavior behind HTTPS proxies.
